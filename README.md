# Friends Reconnected — RAG Chat Widget

Local RAG chatbot using **Playwright + ChromaDB + Ollama + FastAPI**.  
Scrapes real site content, embeds it, and serves a streaming chat widget.

---

## Stack

| Layer | Tool | Why |
|---|---|---|
| Scraping | Playwright + BeautifulSoup | Handles JS-rendered pages, cleans boilerplate |
| Embeddings | sentence-transformers `all-MiniLM-L6-v2` | Fast, local, no API key |
| Vector DB | ChromaDB (persistent) | Local, simple, production-ready |
| LLM | Ollama (Llama 3.2 / Mistral) | Fully local, no API costs |
| API | FastAPI + SSE streaming | Clean REST, streams tokens to frontend |
| Frontend | Vanilla HTML/JS | Single file, embeds anywhere via iframe |

## Setup

### 1. Install Ollama + pull a model
```bash
curl -fsSL https://ollama.com/install.sh | sh
ollama pull llama3.2        # recommended
# ollama pull mistral       # alternative — slightly smarter, slower
```

### 2. Install Python dependencies
```bash
python -m venv venv
source venv/bin/activate    # Windows: venv\Scripts\activate
pip install -r requirements.txt
playwright install chromium  # downloads ~130MB browser
```

### 3. Scrape the site
```bash
# Option A: specific pages
python scrape.py --urls \
  https://friendsreconnected.co.uk/ \
  https://friendsreconnected.co.uk/faq/ \
  https://friendsreconnected.co.uk/testimonials/ \
  https://friendsreconnected.co.uk/people-finding-expert/

# Option B: auto-discover from sitemap
python scrape.py --sitemap https://friendsreconnected.co.uk/sitemap.xml

# Option C: crawl from root (follows all internal links)
python scrape.py --crawl https://friendsreconnected.co.uk/ --max-pages 25
```

### 4. Embed into ChromaDB
```bash
python ingest.py
```

### 5. Start the server
```bash
uvicorn app:app --reload --port 8000
```

### 6. Open the chat
http://localhost:8000

## Adding New Pages Later (Incremental — No Re-Embedding)

### Add a single new page:
```bash
python scrape.py --urls https://friendsreconnected.co.uk/new-page/ --update
python ingest.py
```
`--update` merges the new page with existing scraped data.  
`ingest.py` only embeds NEW or CHANGED chunks (deterministic IDs via upsert).

### Re-scrape if a page's content changed:
```bash
python scrape.py --urls https://friendsreconnected.co.uk/faq/ --update
python ingest.py
```
The scraper detects content changes via fingerprint (MD5 hash).  
Changed pages get re-scraped and re-embedded; everything else is skipped.

### Remove a page:
Delete it from `data/site_content.json`, then:
```bash
python ingest.py --rebuild
```

## Scraper Modes

| Command | What it does |
|---|---|
| `--urls URL1 URL2 ...` | Scrape explicit list of URLs |
| `--file urls.txt` | Scrape URLs from a text file (one per line) |
| `--sitemap URL` | Parse sitemap XML and scrape all discovered URLs |
| `--crawl URL --max-pages N` | Follow all internal links from a root URL |
| `--update` | Merge new pages with existing `site_content.json` |
| `--overwrite` | Re-scrape everything from scratch |

## WordPress Embed

### For local testing with ngrok:
```bash
pip install ngrok
ngrok http 8000
# Copy the https://xxxx.ngrok.io URL
```

### Update the frontend:
Edit `static/index.html`, change:
```js
const API_BASE = "http://localhost:8000";
// → change to your ngrok or production URL
```

### Embed in WordPress:
Pages → New Page → Add Block → **Custom HTML**
```html
<iframe
  src="https://your-url.com"
  width="100%"
  height="620px"
  style="border:none; border-radius:12px; box-shadow:0 2px 12px rgba(0,0,0,0.08);"
  title="Ask Friends Reconnected">
</iframe>
```

## Tuning

**Slower but smarter answers:** increase `TOP_K` in `app.py` (default 4 → try 6)  
**Faster on CPU:** use `llama3.2:3b` instead of default 8B  
**Better chunking:** adjust `CHUNK_SIZE` and `CHUNK_OVERLAP` in `ingest.py`  
**Different LLM:** edit `LLM_MODEL` in `app.py`, then `ollama pull <model>`
